# Phase 2B Project Proof Labeling Report

## Scope

Phase 2B added factual project/proof match labels for city-service money page project cards. The labels clarify whether a visible project is an exact city/service project, a nearby same-service project, a same-city related-service project, or a broader North County related project example.

No project facts, project descriptions, project images, project category routes, slugs, redirects, schema, canonical behavior, robots behavior, or generated `dist/` files were edited.

## Files Changed

- `src/data/projects.ts` — added match-aware money-page helper output and kept `getMoneyPageProjects()` compatibility.
- `src/components/sections/ProjectsSection.astro` — uses match-aware helper only when `moneyPageMode && city && service`, passes labels to project cards, and uses safer copy when related fallback proof is displayed.
- `src/components/ui/ProjectCard.astro` — added optional `matchLabel` prop and subtle text-xs card-body pill rendering only when the prop is passed.
- `audits/phase-2b-project-proof-labeling-report.md` — this validation report.

## Match Types Added

The match-aware helper returns records with:

- `project`
- `matchType`
- `matchLabel`

Allowed match types implemented:

1. `exact-city-service`
   - Used when `project.city === requested city` and `project.service === requested service`.
   - Example label: `Carlsbad water damage project`.

2. `same-service-nearby`
   - Used when the service matches but the project city differs.
   - Example label: `Nearby water damage project`.

3. `same-city-related-service`
   - Used only after exact and same-service matches when the project is in the same city and the service is an approved related water proof service.
   - Example label: `Related water leak project`.

4. `north-county-related`
   - Used only if needed after the previous match tiers for an approved related water proof service outside the requested city.
   - Example label: `North County project example`.

## How Labels Are Derived

Labels are derived only from existing data:

- requested city slug
- requested service slug
- `project.city`
- `project.service`
- display names from existing `cities` and `allServices` data
- a centralized short service-name map for readable labels such as `water damage`, `water leak`, and `flood cleanup`

The helper does not add or change project facts. It does not infer neighborhoods, outcomes, insurance details, timelines, or customer-specific details.
